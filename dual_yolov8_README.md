# Dual-Input Shared YOLOv8n Fine-Tuning

This repo now contains a self-contained PyTorch implementation for the DEEPX dual-camera model preparation stage.

## What It Builds

- Input 1: `front_images`
- Input 2: `top_images`
- Shared fused YOLOv8n trunk copied from `YOLOV8N.onnx`
- Separate `front_head` and `top_head`
- One output class: `person`
- Export names:
  - `shared_backbone.onnx`: `images -> p3, p4, p5`
  - `front_head.onnx`: `p3, p4, p5 -> front_output`
  - `top_head.onnx`: `p3, p4, p5 -> top_output`

The shared trunk reduces duplicated parameters and model memory. It does not remove the need to run convolution work for both camera frames.

## Accuracy Comparison

After saving fine-tuned outputs under `runs/dual_yolov8_person/deepx_split_onnx`, compare them with the original `YOLOV8N.onnx` on the same Roboflow evaluation split:

```bash
python -m dual_yolov8.evaluate_onnx \
  --original-onnx YOLOV8N.onnx \
  --split-onnx-dir runs/dual_yolov8_person/deepx_split_onnx \
  --front-images datasets/roboflow_raw/<front_dataset>/valid/images \
  --front-labels datasets/roboflow_raw/<front_dataset>/valid/labels \
  --top-images datasets/roboflow_raw/<top_dataset>/valid/images \
  --top-labels datasets/roboflow_raw/<top_dataset>/valid/labels \
  --conf 0.001
```

The Colab notebook includes this as the final `Accuracy comparison` section and automatically falls back from `valid` to `test` to `train` if a split is missing.

## Local DEEPX Compile

The DEEPX compiler step is intended to run locally in an Ubuntu environment where DX-COM is installed. The flow follows the DX-COM user manual:

```text
dx_com.compile(model=..., output_dir=..., dataloader=..., calibration_num=100)
```

or the equivalent `dxcom -m model.onnx -c config.json -o output/` CLI form. This project uses the Python API because `front_head.onnx` and `top_head.onnx` each have three feature-map inputs (`p3`, `p4`, `p5`), and DX-COM CLI only supports single-input image calibration.

Prepare about 100 COCO calibration images:

```bash
python -m dual_yolov8.prepare_coco_calibration \
  --out calibration/coco100 \
  --count 100 \
  --imgsz 640 \
  --overwrite
```

Compile the three split ONNX files:

```bash
python -m dual_yolov8.compile_deepx \
  --split-onnx-dir runs/dual_yolov8_person/deepx_split_onnx \
  --calibration-images calibration/coco100 \
  --out-dir runs/dual_yolov8_person/deepx_compiled \
  --calibration-num 100 \
  --imgsz 640 \
  --calibration-method ema \
  --opt-level 1 \
  --gen-log
```

Expected output folders:

```text
runs/dual_yolov8_person/deepx_compiled/shared_backbone/
runs/dual_yolov8_person/deepx_compiled/front_head/
runs/dual_yolov8_person/deepx_compiled/top_head/
```

`shared_backbone.onnx` is calibrated directly with COCO images. The two head ONNX files are calibrated with `p3/p4/p5` tensors generated by running those same COCO images through `shared_backbone.onnx`.

## Colab Flow

When opened from the GitHub Colab badge, the notebook clones or updates this
repository into:

```text
/content/Yolo_Fine_tuning
```

Runtime assets are not committed to git. Put them in Google Drive so the setup
cell can copy them into the cloned repo:

```text
/content/drive/MyDrive/YOLO_optim/
  YOLOV8N.onnx
  calibration/labeled/front
  calibration/labeled/top
```

The setup cell also checks `/content/drive/MyDrive/Yolo_Fine_tuning`,
`/content/drive/MyDrive/Yolo_Fine_tuning_assets`, and
`/content/drive/MyDrive/YOLO_optim_assets`. You can override this with:

```python
%env YOLO_OPTIM_ASSET_ROOT=/content/drive/MyDrive/your_asset_folder
```

Open:

```text
notebooks/deepx_dual_yolov8_finetune_colab.ipynb
```

Set the Colab runtime to GPU, then run cells in order. The notebook now auto-detects these common project roots:

```text
/content/drive/MyDrive/YOLO_optim
/content/drive/MyDrive/Yolo_Fine_tuning
/content/YOLO_optim
/content/Yolo_Fine_tuning
```

If your folder is somewhere else, set this before the project setup cell:

```python
%env YOLO_OPTIM_ROOT=/content/drive/MyDrive/your_folder_name
```

The notebook:

1. Installs dependencies.
2. Mounts Google Drive at `/content/drive`.
3. Resolves `PROJECT_ROOT`.
4. Checks `YOLOV8N.onnx`.
5. Downloads the front-view Roboflow dataset:
   `object-detection-i9fzg/person-detection-tfsgp` version `3`.
6. Downloads the top-view Roboflow dataset:
   `yolo-k9iku/cctv-indoor-person-detection-bi2vp` version `1`.
7. Lets you choose the number of front/top training images.
8. Loads the original YOLOv8n weights into the dual-output model.
9. Fine-tunes `front_head` and `top_head` with a shared trunk, printing validation `P50/R50/AP50` after each epoch when `valid` or `test` splits exist.
10. Exports DEEPX-friendly split ONNX files with opset 17.
11. Runs sequential ONNXRuntime smoke test for `shared_backbone -> front_head/top_head`.

The top-view Roboflow API key is read from Colab Secrets or the environment variable `ROBOFLOW_TOP_API_KEY`.

In Colab, either add a secret named:

```text
ROBOFLOW_TOP_API_KEY
```

or run this before the dataset download cell:

```python
%env ROBOFLOW_TOP_API_KEY=your_key_here
```

The front-view dataset uses the `unauthorized` key value shown in the original Roboflow snippet.

To limit fine-tuning image count in Colab, edit this notebook cell:

```python
MAX_FRONT_SAMPLES = None
MAX_TOP_SAMPLES = None
```

For example:

```python
MAX_FRONT_SAMPLES = 500
MAX_TOP_SAMPLES = 500
```

`None` means use all training images.

## CLI Flow

```bash
python -m pip install -r requirements.txt
python -m dual_yolov8.inspect_onnx YOLOV8N.onnx
python -m dual_yolov8.parity_check --onnx YOLOV8N.onnx
python -m dual_yolov8.train \
  --onnx YOLOV8N.onnx \
  --front-images datasets/roboflow_raw/<front_dataset>/train/images \
  --front-labels datasets/roboflow_raw/<front_dataset>/train/labels \
  --top-images datasets/roboflow_raw/<top_dataset>/train/images \
  --top-labels datasets/roboflow_raw/<top_dataset>/train/labels \
  --front-val-images datasets/roboflow_raw/<front_dataset>/valid/images \
  --front-val-labels datasets/roboflow_raw/<front_dataset>/valid/labels \
  --top-val-images datasets/roboflow_raw/<top_dataset>/valid/images \
  --top-val-labels datasets/roboflow_raw/<top_dataset>/valid/labels \
  --out runs/dual_yolov8_person \
  --warmup-epochs 1 \
  --finetune-epochs 10 \
  --val-every 1 \
  --val-conf 0.25
python -m dual_yolov8.export \
  --checkpoint runs/dual_yolov8_person/best.pt \
  --out-dir runs/dual_yolov8_person/deepx_split_onnx \
  --opset 17
python -m dual_yolov8.verify_split_onnx \
  --checkpoint runs/dual_yolov8_person/best.pt \
  --onnx-dir runs/dual_yolov8_person/deepx_split_onnx
```

## Site Calibration Fine-Tuning

Use the reviewed real-site calibration dataset as the target metric while still
using Roboflow view data for general front/top-view prior:

```bash
python -m dual_yolov8.site_finetune \
  --onnx YOLOV8N.onnx \
  --front-roboflow-root datasets/roboflow_raw/person-detection-3 \
  --top-roboflow-root datasets/roboflow_raw/CCTV-Indoor-Person-Detection-1 \
  --front-calib-root calibration/labeled/front \
  --top-calib-root calibration/labeled/top \
  --out runs/dual_yolov8_site_mix
```

This writes epoch-by-epoch calibration `valid/test` metrics, a test-metric PNG,
`best_val.pt`, `best_test.pt`, and DEEPX-friendly split ONNX files under:

```text
runs/dual_yolov8_site_mix/
```
