# DEEPX NPU 기반 듀얼 카메라 장애물 뒤 객체 인식 시스템 기초 설계 문서

## 1. 시스템 개요

본 시스템은 두 개의 카메라 입력을 사용하여 사용자의 직접 시야에서는 보이지 않는 장애물 뒤 객체를 인식하는 것을 목표로 한다. 하나의 카메라는 사용자 기준 정면 시야를 입력으로 받고, 다른 하나의 카메라는 상단 또는 고정 시점에서 장애물 뒤 영역을 관찰한다.

두 카메라에서 입력된 영상은 NPU 기반 객체 검출 모델을 통해 처리되며, 시스템은 정면 카메라에서 보이지 않는 객체가 상단 카메라에서 검출되는 경우 이를 장애물 뒤 객체로 판단한다.

본 문서는 발표용 설명이 아니라, 구현할 시스템의 기초적인 동작 방식과 모델 구조를 정리하기 위한 문서이다.

---

## 2. 구현 목표

## 2.1 기본 목표

두 개의 카메라로부터 각각 영상을 입력받고, YOLO 기반 객체 검출을 수행하여 장애물 뒤에 존재하는 객체를 인식한다.

## 2.2 시스템이 판단하는 핵심 상황

시스템은 다음과 같은 상황을 기본 대상으로 한다.

1. 정면 카메라에는 객체가 보이지 않는다.
2. 상단 또는 고정 카메라에는 객체가 보인다.
3. 두 카메라의 검출 결과를 비교하여 해당 객체를 사용자의 시야 밖 또는 장애물 뒤 객체로 판단한다.

즉, 본 시스템의 핵심 기능은 문, 출입 여부, 센서 on/off 같은 부가 기능이 아니라, **두 시점의 객체 검출 결과를 이용해 장애물 뒤 객체의 존재를 인식하는 것**이다.

---

## 3. 카메라 구성

## 3.1 정면 시점 카메라

정면 시점 카메라는 사용자가 현재 바라보는 방향의 영상을 입력으로 받는다.

### 역할

- 사용자 기준 전방 시야 입력
- 현재 사용자 화면에 객체가 직접 보이는지 확인
- 정면 시점에서 검출 가능한 객체 출력 생성

### 입력 특성

- 사용자의 실제 시야와 유사한 영상
- 객체가 장애물 뒤에 있으면 검출되지 않을 수 있음
- 실시간 반응성이 중요함

## 3.2 상단 또는 고정 시점 카메라

상단 또는 고정 시점 카메라는 정면 시점에서 보이지 않는 영역을 관찰한다.

### 역할

- 장애물 뒤 영역 관찰
- 정면 시야에서 가려진 객체 검출
- 장애물 뒤 객체 판단을 위한 보조 시점 제공

### 입력 특성

- 정면 시점과 다른 관측 각도
- 상단 또는 고정 위치에서 전체 공간을 더 넓게 볼 수 있음
- 정면 카메라에서 보이지 않는 객체를 검출할 수 있음

---

## 4. 모델 구조

## 4.1 기본 모델 방향

객체 검출 모델은 YOLO 계열 모델을 사용한다. 다만 본 문서에서는 특정 YOLO 버전이나 세부 모델 크기를 지정하지 않는다.

본 시스템은 카메라 두 개에서 각각 입력을 받기 때문에, 단순히 완전히 독립된 모델 두 개를 사용하는 구조가 아니라, 두 입력이 공통 feature extractor를 공유하고 출력부만 각 시점에 맞게 분리되는 구조를 목표로 한다.

---

## 4.2 Shared Backbone 기반 듀얼 입력 구조

두 카메라 입력은 각각 서로 다른 시점을 가지지만, 객체 검출이라는 기본 task는 동일하다. 따라서 영상의 저수준 특징과 객체 공통 특징을 추출하는 앞부분은 공유할 수 있다.

이를 위해 모델은 다음과 같은 구조를 갖는다.

```text
[정면 카메라 입력]
        │
        ├── Preprocess
        │
        ▼
    Shared YOLO Backbone ── Front-view Detection Head ── Output 1
        ▲
        │
        ├── Preprocess
        │
[상단/고정 카메라 입력]
        │
        └────────────────── Top-view Detection Head ──── Output 2
```

위 구조에서 shared backbone은 두 입력에 대해 공통으로 사용되는 특징 추출부이고, head는 각 카메라 시점에 맞게 분리된다.

---

## 4.3 Shared Backbone의 역할

Shared backbone은 두 카메라 입력에서 공통적으로 필요한 시각 특징을 추출한다.

### 주요 역할

- edge, texture, shape 같은 저수준 특징 추출
- 사람 또는 객체의 공통적인 형태 특징 추출
- 두 카메라 모델의 중복 파라미터 감소
- 모델 메모리 사용량 감소 가능성 확보

두 카메라가 서로 다른 시점을 갖더라도 객체 검출에 필요한 기본 시각 특징은 일부 공유될 수 있으므로, backbone을 공유하는 구조는 자원 효율 측면에서 장점이 있다.

---

## 4.4 Camera-specific Detection Head

두 카메라의 시점은 서로 다르기 때문에 출력 head는 각각의 시점에 맞게 별도로 구성한다.

### Front-view Detection Head

정면 시점 카메라에 대응하는 head이다.

역할:

- 사용자 전방 시야에서 보이는 객체 검출
- 일반적인 정면 또는 측면 형태의 객체 인식
- 사용자 시야 내 객체 존재 여부 출력

### Top-view Detection Head

상단 또는 고정 시점 카메라에 대응하는 head이다.

역할:

- 위에서 아래로 내려다보는 시점의 객체 검출
- 정면 시점과 다른 형태로 보이는 객체에 대응
- 장애물 뒤 또는 사각 영역에 있는 객체 출력

---

## 4.5 Fine-tuning 방향

출력 head는 카메라 시점에 따라 fine-tuning을 수행한다.

### 정면 시점 head fine-tuning

정면 카메라 영상에 맞게 학습 또는 보정한다.

- 사람이나 객체가 일반적인 전방 시점으로 보이는 데이터 사용
- 사용자 시야에서 직접 보이는 객체 검출에 최적화

### 상단 시점 head fine-tuning

상단 또는 고정 카메라 영상에 맞게 학습 또는 보정한다.

- 위에서 내려다본 객체 형태를 포함한 데이터 사용
- 정면 시점과 다른 형태의 객체 외형에 대응
- 장애물 뒤 객체 검출 안정성 확보

---

## 4.6 출력 구조

모델은 두 개의 입력에 대해 각각 별도의 출력을 생성한다.

```text
Input 1: 정면 카메라 프레임
Output 1: 정면 시점 객체 검출 결과

Input 2: 상단/고정 카메라 프레임
Output 2: 상단 시점 객체 검출 결과
```

각 출력은 다음 정보를 포함한다.

- 객체 class
- bounding box 좌표
- confidence score
- 해당 카메라 stream ID

이 두 출력은 후처리 단계에서 비교되어 장애물 뒤 객체 존재 여부 판단에 사용된다.

---

## 5. 기본 동작 흐름

## 5.1 전체 처리 순서

```text
1. 정면 카메라와 상단 카메라에서 프레임을 입력받는다.
2. 두 입력 프레임을 모델 입력 형식에 맞게 전처리한다.
3. 두 입력은 shared YOLO backbone을 통해 공통 feature를 추출한다.
4. 정면 카메라 feature는 front-view head를 통과한다.
5. 상단 카메라 feature는 top-view head를 통과한다.
6. 각각 Output 1, Output 2의 객체 검출 결과를 얻는다.
7. 두 출력 결과를 비교한다.
8. 정면 시점에는 없고 상단 시점에는 존재하는 객체를 장애물 뒤 객체로 판단한다.
```

---

## 5.2 장애물 뒤 객체 판단 방식

기초 구현 단계에서는 복잡한 3D 좌표 정합이나 동일 인물 매칭을 수행하지 않는다. 대신 두 카메라의 검출 결과를 비교하여 다음 조건을 만족하는 경우 장애물 뒤 객체로 판단한다.

```text
조건 A: 상단/고정 카메라 Output 2에서 객체가 검출됨
조건 B: 정면 카메라 Output 1에서는 해당 객체가 검출되지 않음
판단: 해당 객체는 정면 시야에서 가려진 장애물 뒤 객체일 가능성이 있음
```

이 방식은 정확한 위치 추정보다는 **가시 여부 차이**를 이용한 단순 판단 구조이다.

---

## 6. 후처리 구조

## 6.1 카메라별 검출 결과 관리

각 카메라의 출력은 별도로 저장한다.

```text
front_detections = Output 1

top_detections = Output 2
```

각 detection은 다음과 같은 형태로 관리할 수 있다.

```text
{
  camera_id: front or top,
  class_id: object class,
  bbox: [x1, y1, x2, y2],
  confidence: score
}
```

## 6.2 기본 비교 로직

가장 단순한 형태의 비교 로직은 다음과 같다.

```text
if top_detections contains object and front_detections does not contain object:
    hidden_object_detected = True
else:
    hidden_object_detected = False
```

초기 구현에서는 이 구조를 통해 시스템이 두 시점의 검출 결과를 비교하고, 장애물 뒤 객체를 인식할 수 있는지 검증한다.

---

## 7. NPU 활용 방식

## 7.1 NPU의 역할

NPU는 YOLO 기반 객체 검출 모델의 추론을 담당한다.

### 수행 작업

- shared backbone 연산
- camera-specific head 연산
- 두 카메라 입력에 대한 객체 검출 결과 생성

## 7.2 CPU의 역할

CPU는 AI 추론 외의 시스템 제어와 후처리를 담당한다.

### 수행 작업

- 카메라 프레임 수집
- 입력 전처리 관리
- NPU 추론 요청
- 두 출력 결과 비교
- 장애물 뒤 객체 판단
- 결과 출력 관리

---

## 8. 구현 범위

## 8.1 본 문서에서 포함하는 범위

본 문서의 기초 구현 범위는 다음과 같다.

1. 두 카메라 입력 수집
2. YOLO 기반 객체 검출
3. shared backbone + camera-specific head 모델 구조
4. 두 카메라별 출력 생성
5. 정면 시점과 상단 시점 검출 결과 비교
6. 장애물 뒤 객체 존재 여부 판단

## 8.2 본 문서에서 제외하는 범위

다음 기능은 본 문서의 기초 동작 방식에서는 제외한다.

1. 문 위치 기반 판단
2. 출입 방향 판단
3. 출입 인원 카운팅
4. PIR 센서 기반 추론 on/off
5. 정확한 3D 좌표 정합
6. 카메라 간 동일 인물 Re-ID
7. 사용자 화면 HUD 세부 구현
8. 특정 YOLO 버전 또는 세부 모델 크기 지정

---

## 9. 기초 시스템 동작 요약

본 시스템은 두 개의 카메라 입력을 사용하여 서로 다른 시점에서 객체를 검출한다. 모델은 YOLO 기반 구조를 사용하며, 두 입력은 공통 shared backbone을 통해 feature를 추출하고, 각 카메라 시점에 맞게 fine-tuning된 별도 detection head를 거쳐 두 개의 출력을 생성한다.

정면 카메라 출력은 사용자 시야 내 객체 검출 결과를 의미하고, 상단 또는 고정 카메라 출력은 정면 시점에서 보이지 않을 수 있는 영역의 객체 검출 결과를 의미한다. 후처리 단계에서는 두 출력 결과를 비교하여 정면 시야에는 없지만 상단 시점에서는 검출되는 객체를 장애물 뒤 객체로 판단한다.
